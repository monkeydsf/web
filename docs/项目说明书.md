# 职策项目说明书

## 1. 项目概述

“职策”是一个面向大学生求职场景的校园求职系统，覆盖学生端、企业端和管理员端三类角色。系统支持岗位浏览、筛选与详情查看，简历维护与附件上传，岗位投递与沟通，站内通知，AI 简历分析，以及后台岗位和背景图管理。

## 2. 项目结构

- `frontend/`：Vue 3 + Vite 前端单页应用
- `backend/`：Spring Boot 3 后端服务
- `database/`：MySQL 建库脚本与初始化脚本
- `docs/`：项目文档、接口说明和生成脚本

## 3. 技术栈

- 前端：Vue 3、Vue Router、Vite、原生 CSS
- 后端：Spring Boot 3.3、Spring Web、Spring Data JPA、Validation
- 数据库：MySQL
- 文件解析：Apache Tika
- AI 服务：阿里云百炼 / DashScope 兼容接口

## 4. 核心功能

### 4.1 学生端

- 注册、登录、退出
- 浏览岗位、按关键词/城市/类型筛选岗位
- 查看岗位详情、任职要求和薪资信息
- 收藏岗位、对比岗位、举报岗位
- 提交岗位投递、查看投递记录、撤回投递
- 与企业在投递对话中沟通
- 维护简历信息
- 上传 PDF / DOC / DOCX 简历附件
- 使用 AI 进行简历分析、岗位匹配和简历修改建议

### 4.2 企业端 / 管理员端

- 登录后台
- 新增、编辑、删除岗位
- 查看全部投递记录
- 修改投递状态
- 安排面试并同步通知
- 查看站内通知
- 查看统计看板
- 管理页面背景图
- 处理岗位举报

### 4.3 AI 工具区

前端工作台提供一组求职辅助工具，包括：

- 改简历
- 找岗位
- 练面试
- 谈 Offer
- 职业性格测试
- 30 天求职规划
- 投递话术生成
- 公司避坑雷达

## 5. 系统架构

系统采用前后端分离模式：

`Vue 页面 -> REST API -> Spring Boot -> JPA -> MySQL`

前端通过 `fetch` 统一访问 `http://localhost:8087/api`，登录 token 保存在 `localStorage`，并通过 `X-Token` 请求头携带。

## 6. 数据库设计

数据库名：`campus_career`

### 6.1 users

存储所有角色账号信息。

- `id`
- `username`
- `password_hash`
- `role`
- `full_name`
- `email`
- `phone`
- `major`
- `graduation_year`
- `created_at`

### 6.2 resumes

每个学生一份简历档案。

- `user_id` 唯一外键关联 `users`
- `expected_city`
- `expected_position`
- `expected_salary`
- `education`
- `skills`
- `project_experience`
- `portfolio_url`
- `file_name`
- `file_url`
- `parsed_text`
- `updated_at`

### 6.3 jobs

岗位信息表。

- `title`
- `company`
- `city`
- `job_type`
- `salary_min`
- `salary_max`
- `education_requirement`
- `description`
- `requirement_text`
- `status`
- `created_at`

### 6.4 job_applications

岗位投递记录表。

- `user_id`
- `job_id`
- `status`
- `cover_letter`
- `applied_at`
- `interview_at`
- `interview_location`
- `interview_note`
- `student_read_at`
- `company_read_at`

### 6.5 其他表

- `job_favorites`：收藏岗位
- `job_reports`：岗位举报
- `notifications`：站内通知
- `interview_schedules`：面试安排
- `application_messages`：投递沟通消息
- `company_profiles`：企业主页信息
- `site_backgrounds`：页面背景图配置

## 7. 主要接口

### 7.1 认证

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`

### 7.2 岗位

- `GET /api/jobs`
- `GET /api/jobs/{id}`
- `POST /api/jobs`
- `PUT /api/jobs/{id}`
- `DELETE /api/jobs/{id}`

### 7.3 简历

- `GET /api/resume`
- `PUT /api/resume`
- `POST /api/resume/file`
- `GET /api/resume/versions`

### 7.4 投递

- `POST /api/applications/jobs/{jobId}`
- `GET /api/applications/mine`
- `GET /api/applications`
- `GET /api/applications/{id}`
- `PUT /api/applications/{id}/status`
- `POST /api/applications/{id}/cancel`

### 7.5 收藏、举报、通知

- `GET /api/favorites`
- `POST /api/favorites/{jobId}`
- `DELETE /api/favorites/{jobId}`
- `POST /api/reports/jobs/{jobId}`
- `GET /api/reports`
- `PUT /api/reports/{id}/status`
- `GET /api/notifications`
- `GET /api/notifications/summary`
- `PATCH /api/notifications/{id}/read`

### 7.6 面试与沟通

- `GET /api/interviews`
- `POST /api/interviews/applications/{applicationId}`
- `GET /api/conversations`
- `GET /api/applications/{applicationId}/messages`
- `POST /api/applications/{applicationId}/messages`

### 7.7 AI 与后台配置

- `POST /api/ai/prompt-test`
- `POST /api/ai/resume-review`
- `POST /api/ai/resume-match/{jobId}`
- `GET /api/dashboard/stats`
- `GET /api/backgrounds`
- `POST /api/backgrounds/{pageKey}`

## 8. 运行说明

### 8.1 后端启动

```bash
cd backend
mvn spring-boot:run
```

后端地址：`http://localhost:8087`

### 8.2 前端启动

```bash
cd frontend
npm install
npm run dev
```

前端地址：`http://localhost:5173`

## 11. 项目亮点

- 三角色权限划分清晰
- 前后端分离，接口职责明确
- 岗位、投递、通知、面试和沟通形成闭环
- 支持简历附件上传与文本解析
- 提供 AI 简历改写和岗位匹配能力
- 支持后台背景图管理和统计看板

## 12. 当前限制

- 简历版本管理目前仅保留单条版本展示
- 权限控制采用 token + 角色校验，后续可升级为更完整的安全体系
- AI 依赖外部服务，接口可用性受密钥和网络影响
