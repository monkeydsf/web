# 职策项目文档

## 1. 需求分析

### 1.1 项目背景

大学生在求职过程中需要集中查看校招岗位、维护个人简历、跟踪投递状态；学校就业管理人员需要发布岗位、审核学生投递并统计就业服务数据。本系统命名为“职策”，围绕校园求职场景，提供一个轻量化的大学生求职管理平台。

### 1.2 目标用户

- 学生用户：浏览岗位、维护简历、投递岗位、查看投递进度。
- 公司人员/管理员用户：发布岗位、维护岗位信息、审核投递记录、查看统计数据。

### 1.3 功能清单

基础功能：

- 用户注册、登录、退出。
- 岗位查询：支持关键词、城市、状态筛选。
- 岗位详情：展示岗位描述、任职要求、薪资和学历要求。
- 简历维护：保存期望岗位、教育经历、技能、项目经历等信息。
- 求职工具台：展示简历状态卡、改简历、找岗位、练面试、谈 Offer 等 AI 核心工具，以及职业性格测试、30 天求职规划、投递话术生成、公司避坑雷达等辅助工具。其中“改简历”支持上传 PDF、DOC、DOCX、TXT 文件，后端调用阿里云百炼 `qwen3.6-flash` 模型生成修改建议。
- 岗位投递：学生可向开放岗位提交投递留言。
- 投递记录：学生可查看或撤回自己的投递。
- 岗位管理：公司人员/管理员可新增、编辑、删除岗位。
- 投递审核：公司人员/管理员可修改投递状态。

进阶功能：

- 条件筛选和关键词搜索。
- 管理员统计看板。
- 后台背景图管理：默认背景图由后端提供，管理员可上传图片替换每个页面背景。
- 学生与管理员权限区分。
- 响应式页面布局。

## 2. 系统设计

### 2.1 总体架构

系统采用前后端分离架构：

```text
Vue 前端页面 -> REST API -> Spring Boot 后端 -> JPA -> MySQL 数据库
```

### 2.2 模块划分

- 用户模块：负责注册、登录、退出和当前用户信息获取。
- 岗位模块：负责岗位列表、岗位详情、岗位新增、编辑、删除。
- 简历模块：负责学生个人简历查询和保存。
- 投递模块：负责岗位投递、投递记录查询、撤回和审核。
- 统计模块：负责管理员数据看板。

### 2.3 核心流程

学生投递流程：

```text
登录 -> 浏览岗位 -> 查看详情 -> 填写投递留言 -> 提交投递 -> 查看投递状态
```

管理员岗位管理流程：

```text
管理员登录 -> 新增岗位 -> 编辑岗位信息 -> 查看学生投递 -> 修改审核状态
```

## 4. 界面原型说明

- 登录注册页：支持学生和公司人员注册，支持学生/公司人员/管理员登录，并显示演示账号。
- 岗位市场页：展示岗位列表、筛选条件、任职要求和投递入口。
- 我的简历页：提供期望岗位、教育经历、技能和项目经历编辑表单。
- 我的投递页：展示岗位名称、企业、投递日期和审核状态。
- 岗位管理页：公司人员/管理员可发布、编辑、删除岗位，并查看统计数据。
- 投递审核页：公司人员/管理员可查看学生投递留言并修改审核状态。
- 背景管理页：管理员可预览各页面背景图并上传新的图片替换。

## 5. 技术说明

前端使用 Vue 3 单页应用，使用 `fetch` 调用后端 REST API，并将登录 token 存储在 `localStorage`。后端使用 Spring Boot 提供接口服务，使用 JPA 进行数据库访问，数据存储在 MySQL。系统通过请求头 `X-Token` 识别用户身份，后台接口会校验公司人员或管理员角色。

## 6. 使用说明

后端启动：

```bash
cd backend
mvn spring-boot:run
```

前端启动：

```bash
cd frontend
npm install
npm run dev
```

浏览器访问 `http://localhost:5173`。演示账号：

- 管理员：admin / admin123
- 学生：student / student123
- 公司人员：company / company123

## 7. 测试与总结

### 7.1 测试用例

| 测试项 | 操作 | 预期结果 |
| --- | --- | --- |
| 登录 | 输入正确账号密码 | 成功进入系统 |
| 注册 | 填写学生信息并提交 | 创建学生账号并登录 |
| 岗位筛选 | 输入关键词或城市 | 展示匹配岗位 |
| 保存简历 | 编辑简历并保存 | 页面提示保存成功 |
| 投递岗位 | 选择开放岗位提交 | 生成投递记录 |
| 岗位管理 | 公司人员或管理员新增岗位 | 岗位列表出现新岗位 |
| 投递审核 | 公司人员或管理员修改状态 | 学生端看到新状态 |

### 7.2 项目亮点

- 功能覆盖学生、公司人员和管理员三类角色。
- 数据库表关系清晰，包含一对一、一对多关系。
- 使用前后端分离架构，接口职责明确。
- 页面支持响应式布局，便于电脑和手机浏览。

### 7.3 不足与优化方向

- 当前文件上传功能未实现，后续可增加简历附件上传。
- 当前鉴权为课程项目轻量实现，后续可升级为 JWT 或 Spring Security 完整权限体系。
- 后续可增加面试邀约、简历附件解析、就业趋势图表等功能。
